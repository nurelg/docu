1. 計画と背景

1.1 ｲﾝﾄﾛﾀﾞｸｼｮﾝ

現在オペル社の提携会社がサービスを供給している範囲では、TIS2000アプリケーションが重要な情報源となっています。  繰り返しDVDで提供されるアップデートにより、技術的資料のほか、テストデバイスやECU（エレクトロニックコントロールユニット）等のソフトウェアが提供されます。

現在TIS2000は改訂中で、新しいアプリケーションとしてインターネット上で利用できるようになります。  これにより、最新の情報が提供できるほか、インストールに関する問題を回避できるなど、明確な利点があります。  インターネットを利用する方向へ進む移行段階として、この新しいアプリケーションをローカルコンピューターで使用できるように、DVDで提供します。  これにより、インターネットが利用できない場合にも問題が軽減されます。

さらに、より速く特定の資料にアクセスできるように、これまでのTIS2000は、情報カテゴリーを圧縮する方向で変更されています。  この変更により、修理や取り外しに関する指示だけでなく、該当するフィールド補修、点検手順、配線図等も含む一体化された目次が表示されるようになります。

2. インストール（サーバーまたは単独コンピューター）

TIS2000同様、このDVDも、サーバーまたはローカルコンピュータ（スタンドアローン）にインストールする必要があります。  しかし、この手順は若干異なっているため、以下に手順を説明いたします。  基本的な違いの1つは、登録が必要であるという点です。  すなわち、GlobalTISは、ユーザーが正規のユーザーであることが確認できるまで、使用できません。  詳しくは、第2.1.4項をご覧ください。

2.1  インストールする前に：

  • インストールや後日アップデートを実行するユーザーには、該当コンピューターの管理者の権限が必要となります。
  • インストール手順を開始する前に、お手元のWindowsシステムが最新のサービスパックでアップデートされていることをご確認ください。
  • GlobalTISをインストールするディレクトリは、存在しないか、空の状態でなければなりません。  このディレクトリの名前には、特殊文字を含めることはできません。
  • USBタイプのハードウェアキー(HWK)がお手元のコンピューターに接続されている場合には、インストール作業中は必ず取り外してください。  これを怠ると、Windowsが自動的にドライバソフトウェアをインストールし、後の作業に支障をきたすことになります。  すなわち、不正なドライバがインストールされることになります。

2.2 インストール中：

  • システムにより、使用する言語を選ぶよう指示されます。 ここで選択する言語は、インストール手順のみに関するものです。 ユーザーインターフェース及び資料の言語設定は、定義方法が異なっています。
  • システムによりポートエントリーを入力するためのプロンプトが表示され、「Transbase Server Port」が既に入力されています。 意図的に変更する場合を除き、変更しないでください。
  • システムによりホスト名を入力するようプロンプトが表示されます。これは、スタンドアローンコンピュータでは「localhost」のままにするか、サーバー上ではコンピュータ名に変更します（デスクトップ上の「マイコンピュータ」を右クリックし、「コンピュータ名」を選択し、「変更」ボタンをクリックすると、正確なコンピュータ名が表示されます）。
  • システムにより「メールサーバー」を入力するようプロンプトが表示されます。  この機能は、フィードバック送信時に、ファックス用に一旦印刷してからではなく、このコンピューターから直接送信する場合に使用するものです。  初期設定では、これらの値を変更する必要はありません。

2.3  インストール終了後、使用開始の前に：

インストール作業が完了したら、ユーザーは手動で「スタート」-「プログラム」-「GlobalTIS」-「GlobalTIS」と進み、アプリケーションを起動する必要があります。

3. 登録：

GlobalTISで作業できるのは、権限のあるユーザーのみです。  登録プロセスによって認証されます。

従来は、HWKを変換するとユーザーはGlobalTISを使用して作業を行うことができました。 シリアルナンバが自動的に権限（ライセンスキー）になりました。　 安全上の理由により、この認証手順は利用不可となり、至急更新する必要があります。

  • オンライン登録
  • マニュアル登録

*)がついた欄は必須項目ですので、必ず必要なデータをご記入ください。  フォームの最後で、ユーザーはオンラインで登録するか、手動で登録するか選択できるようになります。  オンラインで登録する場合には、ユーザー関連データは本部に電子的に送信され、本部でライセンスキーを作成し、返信します。  このライセンスキーはコード化された番号で、「ライセンスキー」欄に入力しなければなりません。

手動で登録する場合には、システムにより、PDF形式の文書が自動的に作成されます。  これは、呈示されるアドレスに添付ファイルとして電子メールで送信していただいても、ファックスで送信していただいてもかまいません。  より迅速に処理できるため、好ましいのは電子メールでの送信です。

3.1  販売メーカーのサブスクリプション

登録過程で、ユーザー毎に1社限定で販売メーカーのサブスクリプションが付与されます。  複数のメーカーからのサブスクリプションが必要な場合には、登録手順を繰り返して、登録を拡張していただくことが必要です。  サブスクリプションを延長した後、新しいアプリケーションのアイコンが表示されるようにするためには、GlobalTISを一旦終了し、再起動する必要があります。

3.2  サブスクリプションの変更

従って、複数のメーカーのサブスクリプションが必要であれば、既存の登録を修正する必要があります。  修正するには、GlobalTISを起動し、タブ「Admin（管理者）」を選択してください。  そこで、新しいサブスクリプションを選択し、適当なボタンを押してマニュアル登録かオンライン登録かを選択してください。

十分なユーザーライセンスがないために、ユーザー数を増やすことが必要であれば、同じ手順を繰り返し実行することが必要です。  この数により、同時にサーバーに接続できるユーザーの数が制限されます。  通常は、この数を修正する必要はありません。

4. GlobalTISのアップデート

新バージョンのGlobalTISがDVDで発行される際には、既にインストールされたプログラムへのアップデートとしてリリースされます。  DVDドライブにDVDを挿入します。  コンピューターの「自動再生」オプションがオンになっている場合には、アップデート手順が自動的に起動されます。  お手元のコンピュータ上でこの機能がオンになっていない場合には、GlobalTIS DVDのルートディレクトリにあるsetup.exeを手動で起動する必要があります。  GlobalTISが、DVDドライブ内のアップデートを自動的に検索することはありません。

5. アプリケーションの起動

このアプリケーションは「スタート」メニューのエントリーをクリックすることで起動できます。  初期設定ではエントリーは、「スタート」-「プログラム」-「GlobalTIS」-「GlobalTIS」で見つかります。

5.1  クライアントの起動

アプリケーションをクライアントコンピューターで起動する場合には、必要なのは「インターネットエクスプローラ」のみです。  アドレスフィールドに、例えば「http://server-name:port/tis2web」といった、サーバーの名前を入力する必要があります。.

例： http://asterix:9080/tis2web

「ポート」は通常「9080」に設定されていますが、変更した場合にはインストール過程で「Tomcat」用のポートを定義したときに入力した値を使用します。
このアドレスをお手元のブラウザのお気に入りに保存しておくと便利です。  GlobalTISは必ずMicrosoft Internet Explorerと併用してください。

5.2 サーバーの起動

サーバーが起動したら、ユーザー名を使ってログインしてください。  初めて起動する際には、ユーザー名を任意に作成して入力してください。  2度目以降は、このユーザー名を選択するか、新しいユーザー名を入力することができます。

5.3  Java Runtime環境

これは以下のモジュールには必須です。

  • ソフトウエアのダウンロード
  • サービス プログラミング システム(SPS)
  • セキュリティアクセス
  • スナップショット
  • Tech2表示
  • テックライン 印刷

お手元のコンピューターにJavaバージョン1.4x以降がインストールされている場合には、GlobalTISモジュールが起動された時点で、Javaアプリケーションが自動的にダウンロードされます。  これは、更新バージョンがサーバー上で利用できるようになったときに1度だけ実行されます。
Javaがまだインストールされていない場合、または上述以前のバージョンの場合には、まずインストールしていただくことが必要です。  Javaバージョン1.5.0_11は「ヘルプ」の以下のセクションからダウンロードできます。 一般 -  アプリケーションのダウンロード

5.4  Javaプロキシ設定

Javaアプリケーションが起動しない場合には、「Java Web Start」アプリケーションのコンフィギュレーションの変更が必要な場合があります。 プロキシ設定を停止することが必要です。
「ファイル」-「環境設定」-「一般」 プロキシに「なし」を選択してください。

5.5  画面設定

ソフトウェアを効率よく、容易に処理できるように、画面の設定を最適化することが重要です。  基本的に、2つのセクションに注目してください。

文書領域の値を適当なサイズに設定します。  「設定」アイコンを選択し、パラメータ「表示領域の高さ」を文書がページ全体に表示されるまで変更してください。

ファンクションキー「F11」を押して、インターネットエクスプローラの表示モードを「フルスクリーン」に変更してください。

「トグル表示」ボタンを押すと、目次の表示モードが切り替えられ、文書の横に表示するか、文書本文のみを表示するか選択することができます。  これにより、利用できる画面スペースを最適化することができます。  トグルボタンは、以下の図の通り、アイコンバーに表示されます。

5.6  言語設定

GlobalTISの言語は、ブラウザの設定で決定されます。  言語を変更するには、ブラウザの「ツール」メニューを開き、「インターネットオプション」から「言語」ボタンを選択してください。  ブラウザの指示に従ってください。
変更を有効にするには、ユーザーはアプリケーションにログオンしなおすことが必要です。

5.7  ブラウザ設定

現行のバージョンは、画面の解像度1024 x 768ピクセル以上、文字のサイズ「小」（インターネットエクスプローラの「表示」-「文字のサイズ」）で最適化されています。  これ以外の設定を利用されると、画面の輪郭にスクロールバーが表示される場合がありますので、スクロールバーを利用して文書全体をご覧ください。

5.8  セッション管理

TIS2000には、「セッション管理」と呼ばれる機能があります。  この機能により、複数のユーザーが独立した環境でTIS2000を利用できます。  車両タイプ、モデルイヤー、検索基準といったすべての設定が保存され、ユーザー同士が干渉することなく、自分が検索した情報を閲覧できます。

GlobalTISでも、同じ技術ではありませんが、同様のメカニズムを備えています。  別のユーザーが今までに定義された設定をリセットせずにシステムを利用する場合には、別のウィンドウでインターネットエクスプローラを開き、独自のIDでログインしてください。


5.9  「進む」及び「戻る」ボタンの機能

インターネットエクスプローラの「進む」及び「戻る」ボタンは、GlobalTISでは機能しません。  従って、これらのボタンを使用してはいけません。  その代わりに、アイコンバーに明確なアイコンがあります。

 

5.10  ソフトウェアのダウンロード、スナップショット表示及びSPS

本質的に、TIS2000で見慣れたモジュールが再利用され、ウェブテクノロジーと一体化されています。  従って、これについての更なる説明は不要だと思われます。

6. サポート

万一ソフトウェアのインストールや操作の際に問題が発生したら、ディーラーアシスタンスセンター(DAC)にお問い合わせください。

7. フィードバック

GlobalTIS実行中にエラーが検出された場合には、容易にご報告いただけるようになっています。  フィードバック用紙が組み込まれており、状況説明が自動的に記入されるようになっていますので、エラーの細かな説明と、ご自分の連絡先情報を記入していただくだけで済みます。  「送信」ボタンをクリックするだけで、自動的に電子メールが作成されます。  もはや別個にメールを作成したり、ファックスを送信したりする必要はありません。

8. アンインストール

万が一GlobalTISをお手元のコンピューターから削除される場合には、2通りの方法があります。

  1. 専用のアンインストール手順は、「スタート」-「プログラム」-「GlobalTIS」-「GlobalTISのアンインストール」で起動できます。
  2. また、「スタート」-「設定」-「コントロールパネル」-「プログラムの追加と削除」で削除することも可能です。  エントリー「GlobalTIS」を探し、「変更/削除」ボタンをクリックしてください。

以前のインストールコンフィギュレーションに関して少数の情報がシステム内に残ります。   これは、登録手順の間に入力および受信した情報などです。   これらは再利用でき、GlobalTISを新しくインストールする際に、手順が楽になります。   新しくGlobalTISをインストールする際に、「Ignore old settings（今までの設定を無視する）」を選択すると、登録データが削除されるため、ライセンス手順をもう一度実行することが必要になります。

 

9. TIS2000 との相違点

  • 操作は今まで以上にアイコンで制御されるようになっています。
  • 車両の詳細は、TIS2Webインストール後に一度定義すれば、それ以上定義する必要はありません。
  • 起動ページには、TIS2000で慣れておられるほどのアイコンはありません。
TIS2Webを起動して、右上隅の「設定」アイコンをクリックし、利用可能なエントリーから1つを選択してください。   プログラムを終了しても、この設定は残ったままとなります。

より重要な提案は、右上隅の「ヘルプ」機能で見つけることができます。
